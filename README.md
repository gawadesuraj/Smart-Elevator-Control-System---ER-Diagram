# Smart Elevator Control System - ER Diagram

## Overview

This project contains the Entity Relationship Diagram (ERD) for a smart elevator control platform used in large commercial buildings such as corporate towers, malls, airports, hospitals, and residential high-rises. The system manages multiple buildings, elevators, floor requests, ride assignments, status tracking, and maintenance history.

The design focuses on separating elevator configuration from real-time ride activity and operational tracking.

---

## System Workflow

Floor Request → Elevator Assignment → Ride Execution → Ride Log
→ Status Update → Maintenance Tracking (if needed)

## Entities

### Buildings

Stores building information.

* building_id (PK)
* name
* location
* total_floors

---

### Floors

Represents floors inside a building.

* floor_id (PK)
* building_id (FK)
* floor_number

---

### Shafts

Represents elevator shafts.

* shaft_id (PK)
* building_id (FK)
* shaft_number

---

### Elevators

Stores elevator configuration.

* elevator_id (PK)
* building_id (FK)
* shaft_id (FK)
* elevator_number
* capacity

---

### Elevator Floors

Maps elevators to floors they serve.

* id (PK)
* elevator_id (FK)
* floor_id (FK)

---

### Requests

Floor requests generated by users.

* request_id (PK)
* building_id (FK)
* source_floor_id (FK)
* destination_floor_id (FK)
* direction
* requested_at
* status

---

### Ride Assignments

Maps request to elevator.

* assignment_id (PK)
* request_id (FK)
* elevator_id (FK)
* assigned_at

---

### Ride Logs

Stores completed ride data.

* ride_id (PK)
* elevator_id (FK)
* request_id (FK)
* start_time
* end_time
* status

---

### Elevator Status

Tracks current elevator state.

* status_id (PK)
* elevator_id (FK)
* status
* updated_at

Examples:

* idle
* moving
* maintenance
* offline

---

### Maintenance

Stores maintenance history.

* maintenance_id (PK)
* elevator_id (FK)
* issue
* status
* started_at
* completed_at

---

## Relationships

* One building contains multiple floors
* One building contains multiple elevators
* One shaft contains one elevator
* One elevator serves multiple floors
* One floor served by multiple elevators
* Requests generated from floors
* Requests assigned to elevators
* Elevators complete multiple rides
* Elevator status tracked separately
* Maintenance tracked per elevator
